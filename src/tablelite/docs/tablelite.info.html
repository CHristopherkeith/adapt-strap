### The column definition object is an array of definition objects ### The definition object should have properties shown in the example below: { columnHeaderDisplayName: 'String', // Not Required if columnHeaderTemplate is specified columnHeaderTemplate: 'String', // Not Required if columnHeaderDisplayName is specified. displayProperty: 'String', // Not required if templateUrl is provided. cellFilter: 'String', // - Optional - the filter expression to apply to the displayProperty templateUrl: 'String', // - Not required if displayProperty is specified - look at artistPicture.html, sortKey: 'String', // - Optional - specified key (in most cases: same as displayProperty) will be used for sorting width: 'String' // - Optional - the explicitly specify width of the column. (Ex: '10em', '30px'). If not specified, width is calculated automatically based on content width. }
